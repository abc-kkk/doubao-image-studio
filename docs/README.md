# 豆包生图助手 产品文档

## 一、产品概述

### 1.1 是什么

豆包生图助手是一款桌面端工具，通过 Chrome 浏览器扩展劫持豆包网页的 AI 生图接口，将生成能力"移植"到本地桌面端进行管理和展示。

**核心价值**：让用户享受豆包 AI 生图能力的同时，获得原生桌面级的画廊管理、图片压缩和本地持久化体验。

### 1.2 解决的问题

| 用户痛点 | 解决方案 |
|---------|---------|
| 豆包网页版没有画廊管理 | 桌面端自动保存所有生成记录 |
| 无法本地保存高清无水印图 | 一键下载原图到本地 |
| 参考图功能使用不便 | 画廊支持一键设为参考图 |
| 批量处理图片麻烦 | 内置图片压缩工具 |
| 需要维护 Node.js 环境 | 内置 Rust 服务端，零依赖 |

### 1.3 适用人群

- 需要管理大量 AI 生成图片的设计师/创作者
- 希望本地备份 AI 生成内容的用户
- 需要图片压缩工具的办公人群

## 二、功能介绍

### 2.1 核心功能

#### 图片生成
- **文字生图**：输入 Prompt，生成图片
- **图生图**：上传参考图，AI 基于参考图生成新图
- **比例控制**：支持多种图片比例（1:1、16:9、9:16 等）
- **批量生成**：支持同时生成多张图片

#### 画廊管理
- **历史记录**：自动保存所有生成记录（Prompt、图片、时间）
- **一键复用**：点击历史记录快速复用 Prompt
- **设为参考图**：将历史图片一键设为参考图
- **搜索过滤**：支持关键词搜索历史记录
- **批量预览**：网格视图快速浏览

#### 图片处理
- **图片压缩**：支持 JPEG/PNG/WebP 压缩
- **质量调节**：可设置目标文件大小
- **实时预览**：压缩前后对比

#### 下载管理
- **原图下载**：下载高清无水印原图
- **自定义路径**：可设置下载保存目录

### 2.2 扩展功能

#### Chrome 浏览器扩展
- 劫持豆包页面 AI 生图接口
- 转发生成结果到桌面端
- 实时显示生成进度
- 自动保存生成记录

## 三、工作原理

### 3.1 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                        用户界面 (React)                       │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   画布区域   │  │   画廊面板   │  │     图片压缩工具     │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└───────────────────────────┬─────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                     Tauri 桌面端 (Rust)                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              内置 HTTP/WebSocket 服务                  │   │
│  │   • REST API (端口 8081)                              │   │
│  │   • WebSocket (端口 8081/ws)                          │   │
│  │   • SQLite 数据库                                      │   │
│  │   • 图片处理 (压缩/代理)                               │   │
│  └─────────────────────────────────────────────────────┘   │
└───────────────────────────┬─────────────────────────────────┘
                            │ WebSocket
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                   Chrome 浏览器扩展                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │Background WS│  │Content Script│  │    Hook (劫持)      │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└───────────────────────────┬─────────────────────────────────┘
                            │ 劫持豆包 API
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                       豆包网页 (doubao.com)                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              AI 生图接口 (被劫持)                      │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 数据流

1. 用户在桌面端输入 Prompt 或上传参考图
2. 桌面端通过 WebSocket 发送生成请求到 Chrome 扩展
3. Chrome 扩展通过 Hook 操控豆包网页触发 AI 生图
4. 豆包返回流式响应，扩展拦截并转发给桌面端
5. 桌面端展示图片，并保存到 SQLite 数据库

### 3.3 技术栈

| 层级 | 技术 |
|------|------|
| 桌面端 UI | Tauri v2 + React 19 + TypeScript + Tailwind CSS v4 |
| 状态管理 | Zustand |
| 桌面端内核 | Rust (Tauri) |
| 内置服务端 | Rust (Axum + Tokio + Tungstenite) |
| 数据库 | SQLite (rusqlite) |
| 图片处理 | Rust (image crate) |
| Chrome 扩展 | TypeScript + Vite CRX (Manifest V3) |

## 四、安装指南

### 4.1 系统要求

| 要求 | 说明 |
|------|------|
| 操作系统 | macOS 10.15+ / Windows 10+ |
| 浏览器 | Chrome 88+ / Edge 88+ |
| 内存 | 建议 4GB+ |
| 磁盘 | 预留 500MB+ |

### 4.2 安装步骤

#### 第一步：安装桌面端应用

1. 访问 [Releases](https://github.com/abc-kkk/doubao-image-studio/releases) 页面
2. 下载最新版本的 DMG（macOS）或 EXE（Windows）安装包
3. macOS：双击 DMG，将应用拖入 Applications 文件夹
4. Windows：双击 EXE，按提示完成安装

#### 第二步：安装 Chrome 扩展

1. 访问 [Releases](https://github.com/abc-kkk/doubao-image-studio/releases) 页面
2. 下载 `doubao-assistant-chrome-extension.zip`
3. 解压到任意目录（如桌面）
4. 打开 Chrome，访问 `chrome://extensions/`
5. 开启右上角的"开发者模式"
6. 点击"加载已解压的扩展程序"
7. 选择解压后的 `dist` 文件夹

#### 第三步：验证安装

1. 启动桌面端应用
2. Chrome 扩展图标应显示"Connected"状态
3. 打开 [豆包官网](https://www.doubao.com/)，确认扩展正常工作

## 五、使用指南

### 5.1 文字生图

1. 启动桌面端应用，确保连接状态为"Connected"
2. 在底部 Prompt 输入框输入描述词
3. 选择图片比例
4. 点击发送按钮
5. 等待图片生成完成，自动保存到画廊

### 5.2 图生图

1. 在画廊中点击任意历史图片
2. 点击"设为参考图"按钮
3. 输入新的描述词
4. 点击发送，AI 将基于参考图生成新图

### 5.3 图片压缩

1. 点击左侧导航栏"图片压缩"
2. 拖拽或选择本地图片
3. 设置目标格式和质量
4. 预览压缩效果
5. 点击下载保存

### 5.4 画廊管理

- **浏览**：网格视图查看所有历史图片
- **搜索**：输入关键词搜索
- **复用 Prompt**：点击历史项快速填入 Prompt
- **删除**：点击删除按钮移除记录

## 六、常见问题

### Q1: 连接状态一直显示"Connecting"

**原因**：桌面端应用未启动或扩展无法连接

**解决方法**：
1. 确认桌面端应用已启动
2. 重启桌面端应用
3. 在扩展页面点击扩展图标，查看错误信息
4. 确认端口 8081 未被其他程序占用

### Q2: 生成的图片显示不出来

**原因**：可能是网络问题或豆包接口变更

**解决方法**：
1. 刷新豆包网页
2. 重启桌面端应用和扩展
3. 检查豆包账号是否登录

### Q3: 如何更新扩展？

1. 从 Releases 页面下载新版 zip
2. 解压覆盖原目录
3. 在扩展管理页面点击刷新按钮

### Q4: 数据保存在哪里？

- 图片元数据：SQLite 数据库（应用数据目录下）
- 配置文件：应用设置存储
- 下载的图片：用户选择的下载目录

### Q5: 可以同时生成多张图吗？

支持。连续点击发送按钮可以排队生成多张图片。
